📌 Smart Home API (FastAPI + MongoDB)
🧠 توضیح پروژه

این پروژه یک API ساده اما کاربردی برای مدیریت دستگاه‌های خانه هوشمند (Smart Home) است که با استفاده از FastAPI و MongoDB ساخته شده است. هدف پروژه پیاده‌سازی مفاهیم اصلی بک‌اند مثل احراز هویت، مدیریت کاربران، امنیت API و ارتباط بین کاربر و داده‌های او است.

🚀 تکنولوژی‌های استفاده شده
Python
FastAPI
MongoDB (Motor async driver)
JWT Authentication
Passlib (bcrypt برای رمزنگاری پسورد)
Pydantic (برای validation داده‌ها)
🔐 بخش Authentication

در این پروژه سیستم احراز هویت کامل پیاده‌سازی شده است:

ثبت‌نام کاربر (Register)
ورود کاربر (Login)
تولید JWT Token
بررسی توکن برای دسترسی به APIهای محافظت‌شده

پسورد کاربران به صورت امن با الگوریتم bcrypt هش (hash) می‌شود و در دیتابیس ذخیره می‌گردد.

👤 مدیریت کاربران
ساخت کاربر جدید
دریافت اطلاعات کاربر فعلی با استفاده از JWT (/auth/me)
ذخیره کاربران در MongoDB
📱 مدیریت دستگاه‌ها (Devices)

در این بخش کاربران می‌توانند دستگاه‌های خود را مدیریت کنند:

افزودن دستگاه جدید
مشاهده لیست دستگاه‌های مربوط به هر کاربر
حذف دستگاه

تمام دستگاه‌ها به یک owner_id متصل هستند که مشخص می‌کند هر دستگاه متعلق به کدام کاربر است.

🔒 سیستم امنیت و دسترسی
استفاده از JWT برای احراز هویت
استفاده از Depends در FastAPI برای کنترل دسترسی
هر کاربر فقط به داده‌های خودش دسترسی دارد
جلوگیری از دسترسی کاربران به اطلاعات دیگران (Ownership Control)
🗄️ ساختار دیتابیس
users

شامل اطلاعات کاربر:

name
phone
password (hashed)
devices

شامل اطلاعات دستگاه:

name
device_type
owner_id (شناسه کاربر)
🧩 ساختار پروژه
⚙️ ویژگی‌های مهم پروژه
API کاملاً async
اتصال به MongoDB با Motor
امنیت با JWT
رمزنگاری پسورد
معماری ماژولار (Router-based)
کنترل دسترسی سطح کاربر
🎯 هدف آموزشی پروژه

این پروژه برای تمرین و یادگیری مفاهیم زیر طراحی شده است:

ساخت REST API با FastAPI
کار با دیتابیس NoSQL (MongoDB)
Authentication و Authorization
طراحی ساختار backend واقعی
مدیریت stateless authentication با JWT
💡 نتیجه نهایی

این پروژه یک نمونه ساده از backend واقعی سیستم‌های IoT / Smart Home است که در آن هر کاربر به صورت ایزوله داده‌های خودش را مدیریت می‌کند.
